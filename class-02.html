Class 02 (Wednesday 06/03/2024)

Introduction

The focus of today’s class is the mise-en-place for your thesis publication: setting up your documents, master spreads and stylesheets, in a manner that will make your design process more efficient and consistent.

Obviously there are numerous schools of thought around graphic design principles, and not just one right way of designing a book or coding a website. Below you find some steps and tips that will help you get started building a system within which you can play around with layouts and styling.

Requirements

Make sure you have the necessary software to design or code your publication.

→ print

Adobe Indesign (this is what will be used in this tutorial) or suitable alternative.

→ code

A code editor, such as Visual Studio Code or Sublime.

1. A blank canvas

→ print

Open a new Indesign file. Enter the dimensions of your page. Generally leave ‘facing pages’ ticked, and turn on ‘primary text frame’. You can already specify the margins and columns to the layout you have in mind, but this can also be done later.

In case you plan to have anything that flows over the edge of the paper (e.g. images or printed background colour) add a bleed, which an extended area that will be cut off.

The slug is another (optional) extra area beyond the bleed that can hold printing information, customized color bar information, or display other instructions and descriptions for other information in the document.

→ code

Create a folder on your computer, open this folder with your code editor and create a new HTML (HyperText Markup Language) file called ‘index.html’. To start setting up the file, you can copy the code from this empty page (right click > View Page Source).

To view your HTML file in its web form simply open it in your web browser and refresh when you have made an update to the code (since it is still local on your computer you don’t need to be connected to the internet).

The size of your canvas is fluid and can flow beyond the ‘viewport’, the user's visible area of a web page, both horizontally and vertically. It is useful to take the viewport as a measure to base your layout on, for instance by taking the full ‘view-width’ as the horizontal limit of your page.

You can set this up through a CSS (Cascading Style Sheets) file that is linked to the page in the ‘head’ (see style.css in the example page). Here the ‘body’ element’s width is set to 100vw, which means 100% of the viewport’s width.

2. Base(line)

A good base from which to start specifying the measurements of your layout is actually the body text. Add a bit of a content, whether the actual content of your thesis text or some Lorem Ipsem. Set it in your desired typeface and try out some fontsizes and lineheights.

Generally in books the main body text is between 10 and 12 points and for websites 16 pixels is a good starting point. Standard lineheight is the fontsize x 1.2 (120%), but you can make this slightly smaller or bigger to your liking.

Keep in mind that optimal line length for body text is 50–75 characters (it’s good to anticipate the approximate width of your textarea here).

→ print

From the ideal lineheight of your body text you are going to establish a baseline grid. A baseline grid consists of horizontal guide lines you can align the text throughout your entire document to. You can also base other measurements off of this value, such as the margins and gutters within a grid.

Through Preferences > Grids... you can specify the baseline grid. To have a perfectly distributed baseline grid across the entire height of your page, change Start to 0 and use this tool called fitbaseline to find a fitted value nearest to your desired lineheight. Enter this value for Increment Every.

Make sure ‘Align to baseline grid’ is selected for your paragraph. You may have to decrease its lineheight to not have it jump to the next line.

→ code

Set the desired typeface through the font-family property. If you want to use one that is not a system/web-safe font, add it using the @font-face rule (explained here).

In code you don’t have to mind the alignment of your text through a baseline grid. However you can use the font-size of the main body text as a base value from which to determine other measurements, by adding ‘html { font-size: 16px;}’ at the top of your CSS file. You could even make this root font-size responsive to the view-width by making its value for instance ‘1.2vw’.

Now you can refer back to this for other properties such as margins, paddings or heading sizes using rem units (root em). 1rem equals the root font-size.

3. Margins and grids

Unless you have very experimental plans for your design that require a different approach, working with a grid of columns and rows is essential for creating a coherent layout throughout your publication.

→ print

In File > Document Setup edit the margin, preferably using our fitted baseline value. Generally in books the margins are not the same in size, but are larger on the inside and bottom. For instance you can set the top and outside to 2x the baseline, bottom to 4x baseline and inside to 6x baseline. You’ll see how in this way the margin aligns nicely to your baseline grid.

Now make sure to go into your Master spread, select both pages and go to Layout > Margins and Columns. Depending on your desired layout, set the amount of columns. Set the gutter to the baseline value.

For the rows of your grid, which can be used for placing images or text blocks, go to Layout > Create Guides. To also align this perfectly to the grid, count the lines in the area and find a whole number that it is divisible by (and makes sense as an amount of rows). Adjust your top and bottom margins accordingly if needed. For instance an area of 45 lines can be divisible by 5. Five rows need four gutters, so take one line from the area and add it to your bottom margin.

→ code

Add a ‘main’ element and give it a margin such as 1rem.

The default layout in an HTML page is elements that stack underneath each other as blocks. One way to create a simply layout in which elements can sit next to each other is through the grid layout module (explained here).

In this example thesis page you can right click > Inspect to look at how the grid and ‘grid-column-start’ and ‘grid-column-end’ properties can be used to place different elements across specific columns.

Keep in mind that the amount of things that can exist horizontally next to each other on a desktop computer is different than on a small mobile device. Usually it’s best to go back to ‘display: block;’ for the latter. Scroll down the example page’s stylesheet to see how this is altered for mobile screens.

4.Typographic styles

Earlier you determined the size of the main body text. To create hierarchy in terms of headings, subheadings and other types of text, you need to differentiate through size, color, weight, case (e.g. capitalisation), position and/or aligment.

→ print

To this efficiently throughout the entire document, and easily make changes when needed, use Paragraph Styles for each type of heading, subheading or paragraph.

Character Styles are used to create specific styling for individual characters, words or sentences within a paragraph. For instance to italicize the title of a book you are referencing.

→ code

In HTML there is a set hierarchical semantic structure for headings (h1-h6) and paragraphs (p), which is good to adhere to for accessiblity reasons. Still you can style these elements however you want in the CSS, in a similar manner to paragraph styles in Indesign, and create additional classes for different types of paragraphs.

Similar to Character Styles in Indesign, you can use semantic elements such as the ‘i’ tag or ‘span’ tag for inline styling.

5. Orientation/navigation

A standard section or page present in a publication is of course the table of contents. Next to this you can also think of other ways to help your reader navigate or orient themselves throughout your thesis.

→ print

Page numbers are obviously the most important element for navigation in a printed publication. You add these in your Master spread, for instance on the bottom at an equal distance from the edge of the page as the top-margin. Select the number and go to Type > Insert Special Character > Markers > Current Page Number.

You can also add a running head, with the respective chapter title, to your Master spread.

→ code

On a website you can make use of (internal) hyperlinks to jump through the thesis. In the example thesis you can see how the table of contents consist of ‘a’ (anchor) tags that when clicked have you scroll to the section on the page with the matching ID.

Furthermore, you can make the table of contents ‘fixed’ or ‘sticky’ to remain in place when you scroll.

6. Images

→ print

Add images to the grid by using the full width of one of more columns or leaving that as whitespace in which to place the image, or have it run across the full page or spread.

If the image overlaps with a textframe use Text Wrap to e.g. have the text go around and jump to the next line or column.

→ code

Add images through an ‘img’ tag with a ’figcaption’ nested together inside a ‘figure’. In the example thesis they are place in between paragraphs, appearing to the right of the text that supposedly refers to it, and sticking to the top as you scroll down untill the section is over.

Make sure ‘img { width: 100%;}’ is added to your stylesheet to have the image fit the full width of the figure. You can also work with height and object-fit, or play with properties like filters or border-radius for rounded edges and circular forms (explained here).

The digital equivalant of the full page/spread image can be done through a background-image property (explained here).

7. References

Decide whether you want to references to be shown as foot-, side- or endnotes. As the terms suggest footnotes appear at the bottom of the page, sidenotes to the side of a text and endnotes at the end of the publication or section/chapter of a publication.

→ print

To insert a footnote go to Type > Insert Footnote, and alter the styling of it in Type > Document Footnote Options. Likewise for endnotes. For sidenotes you can make use of anchored frames (explained here).

→ code

Similarly to the internal hyperlinks in the table of contents, you can link back and forth between the number of your foot-/side-/endnote and the reference, like in the example thesis.
